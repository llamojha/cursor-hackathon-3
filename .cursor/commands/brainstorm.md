Generate creative ideas and solutions for any topic, problem, or challenge.

## Input
- Describe your topic, problem, or goal
- Reference files with @filename for context
- Say "project context" to auto-scan the current codebase

## Process
1. **Understand**: Clarify the challenge and constraints
2. **Diverge**: Generate diverse ideas without judgment
3. **Connect**: Find patterns and build on concepts
4. **Evaluate**: Assess feasibility and impact (this is a ~2.5h hackathon — bias toward demoable in the time available)

## Output Format
### Quick Ideas (5-10 rapid-fire concepts)
- Bullet list of initial thoughts

### Developed Concepts (3-5 detailed ideas)
For each concept:
- **Idea**: Clear description
- **Why it works**: Key benefits / demo appeal
- **Implementation**: How to execute with the current stack (libreyolo-web, webcam, canvas)
- **Considerations**: Potential challenges / time risk

### Next Steps
- Most promising direction to explore
- Questions to investigate further

## Constraints
- Prefer ideas that run fully in-browser (WebGPU/WASM) — no backend to demo
- Account for the time limit; favor a sharp single feature over breadth
- Build on existing project context when available
